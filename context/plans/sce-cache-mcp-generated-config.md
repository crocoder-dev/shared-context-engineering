# Plan: sce-cache-mcp-generated-config

## Change summary

Add canonical Pkl-authored MCP configuration for the existing `sce mcp` stdio server so all generated client profiles wire in the Smart Cache Engine automatically. The change must cover manual OpenCode, automated OpenCode, and Claude Code using each client's documented MCP configuration shape instead of ad hoc post-generation edits.

## Success criteria

- OpenCode's canonical Pkl sources emit a generated MCP config artifact that registers a local `sce` MCP server using the `sce mcp` command for both `config/.opencode/**` and `config/automated/.opencode/**`.
- Claude Code's canonical Pkl sources emit the documented project-shared MCP configuration surface for the same local `sce` stdio server, with `sce` as the server name and `sce mcp` as the launched command.
- Generated-output ownership and workflow docs clearly describe the new MCP config artifacts so future edits stay in canonical Pkl sources instead of generated files.
- Regeneration, parity, and repo validation checks pass for the new generated artifacts.

## Constraints and non-goals

- In scope: canonical Pkl sources, generator wiring, generated config artifacts, and durable context updates required by the new cross-target MCP contract.
- In scope: all OpenCode profiles currently generated by this repo plus Claude Code project configuration.
- Constraint: preserve the current CLI contract of `sce mcp`; this plan configures existing behavior and does not expand the MCP server's runtime feature set.
- Constraint: use each client's documented local/stdin MCP schema rather than inventing repo-specific config shapes.
- Out of scope: remote MCP servers, OAuth/auth flows, additional MCP tools, client-specific interactive setup commands, and unrelated cleanup in generated config trees.

## Assumptions

- The MCP server identifier is `sce` across all targets unless an existing generated-config constraint forces a target-specific alias.
- The launched command is exactly `sce mcp`, with no extra environment variables unless a client requires them for schema validity.
- Claude Code should use its documented project-shared MCP configuration surface (expected to be `.mcp.json` unless implementation evidence shows a different canonical generated target is required).
