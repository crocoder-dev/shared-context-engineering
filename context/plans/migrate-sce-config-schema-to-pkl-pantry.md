# Plan: Migrate SCE Config Schema to pkl-pantry org.json_schema

## Change summary

Rewrite `config/pkl/base/sce-config-schema.pkl` to use the typed `org.json_schema.JsonSchema` module from Apple's pkl-pantry instead of hand-crafted `Dynamic` objects. This improves type safety, maintainability, and aligns with the Pkl ecosystem's canonical JSON Schema authoring approach.

## Success criteria

1. `config/pkl/base/sce-config-schema.pkl` uses `org.json_schema.JsonSchema` module from pkl-pantry
2. Generated `config/schema/sce-config.schema.json` is byte-for-byte identical to current output
3. `nix run .#pkl-check-generated` passes (parity check)
4. `nix flake check` passes (all validation)
5. PklProject dependency declaration follows pkl-pantry conventions

## Constraints and non-goals

**In scope:**
- Add `PklProject` and `PklProject.deps.json` for `org.json_schema` dependency
- Rewrite `sce-config-schema.pkl` to use typed `JsonSchema` module
- Preserve exact JSON Schema output (byte-for-byte parity)

**Out of scope:**
- Changes to the JSON Schema structure or semantics
- Changes to `bash-policy-presets.pkl` (already uses typed Pkl classes)
- Changes to CLI config validation logic
- Adding new schema features

**Constraints:**
- Must use pkl-pantry package URI: `package://pkg.pkl-lang.org/pkl-pantry/org.json_schema@<version>`
- Must work with Nix-provided Pkl (no host-level install required)
- Generated output must match existing schema exactly

### T02: Rewrite sce-config-schema.pkl to use org.json_schema.JsonSchema

**Task ID:** T02

**Goal:** Convert `config/pkl/base/sce-config-schema.pkl` from `Dynamic` object construction to the typed `org.json_schema.JsonSchema` module API.

**Boundaries (in/out of scope):**
- In: Import `org.json_schema.JsonSchema` module
- In: Use `amends` pattern to extend JsonSchema module
- In: Convert all `Dynamic` objects to typed property assignments
- In: Handle `allOf`/`not` constraints for mutually exclusive presets
- In: Handle `dependentRequired` constraint
- Out: Changes to `bash-policy-presets.pkl` (already typed)
- Out: Changes to generated JSON output structure

**Done when:**
- `config/pkl/base/sce-config-schema.pkl` uses `amends` pattern with `org.json_schema.JsonSchema`
- All schema properties use typed API (`type`, `properties`, `required`, `enum`, etc.)
- Generated `config/schema/sce-config.schema.json` is byte-for-byte identical to current version
- `nix develop -c pkl eval config/pkl/generate.pkl` succeeds

**Verification notes:**
```bash
# Generate and compare
nix develop -c pkl eval -m . config/pkl/generate.pkl
git diff config/schema/sce-config.schema.json

# Should show no diff if output is identical
```

**Implementation approach:**
1. Import: `import "package://pkg.pkl-lang.org/pkl-pantry/org.json_schema@1.1.0#/JsonSchema.pkl"`
2. Change module from `module` to `amends "package://...#/JsonSchema.pkl"`
3. Set top-level properties: `$schema`, `$id`, `title`, `description`, `type`, `additionalProperties`
4. Use `properties { ["name"] { ... } }` syntax for object properties
5. Use `allOf` and `not` for mutually exclusive preset constraints
6. Use `dependentRequired` for `log_file_mode` -> `log_file` dependency

**Key conversions from Dynamic to typed:**
```pkl
# Before (Dynamic):
new Dynamic {
  ["type"] = "string"
  ["enum"] = new { "error"; "warn"; "info"; "debug" }
}

# After (typed):
type = "string"
enum = new { "error"; "warn"; "info"; "debug" }
```
